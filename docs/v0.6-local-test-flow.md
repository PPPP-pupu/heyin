# Heyin v0.6 — Local MVP Test Flow

This document describes the complete end-to-end test flow for the v0.6 Local MVP. All data stays in the browser (localStorage + IndexedDB). No server, no accounts, no network required.

## Prerequisites

- A modern browser (Chrome/Firefox/Edge/Safari) with microphone access enabled.
- Run `npm run dev` from the project root.
- Open `http://localhost:3000`.

---

## 1. Create a Project

1. From the home page, click **"Create Chorus Project"**.
2. Fill in:
   - **Project Title**: e.g. "十年大合唱"
   - **Song Name**: e.g. "十年"
   - **Lyrics**: Paste multi-line lyrics (one phrase per line).
   - **Voices per Line**: 3 (default).
3. Click **"Create Project"**.
4. You are redirected to the project detail page.
5. Verify: the project title, song name, lyric lines, and empty voice slots are all displayed.

**Acceptance:** Project appears in the `/explore` list. Demo project at `/project/demo` is pre-seeded.

---

## 2. Join a Project (as a Participant)

1. Open the demo project at `/project/demo`.
2. Click **"Share"** to copy the join URL (`/join/demo`).
3. Open a new browser tab, paste the URL.
4. The **Nickname Gate** appears. Enter a nickname (e.g. "小雨").
5. Click **"Join Chorus"**.
6. Verify:
   - Progress bar shows filled/total voice count.
   - "Joined as 小雨" is displayed.
   - Lyric lines with filled (playable) and empty (selectable) slots are shown.
   - Click **"change"** on the guest card to rename.

**Acceptance:** Guest profile is persisted in localStorage (`heyin:guest`). After page refresh, nickname is remembered.

---

## 3. Record and Submit a Voice

1. On the Join page, click an **empty slot** (dashed border, "+" icon).
2. The slot changes to **claimed** status (amber pulse, "You are recording...").
3. The **RecordingModal** opens, showing the lyric text.
4. Click **"Start Recording"** — speak into your microphone.
   - Red pulsing dot indicates active recording.
   - Timer counts up.
5. Click **"Stop Recording"**.
6. Preview the recording using the built-in audio player.
7. Fill in **Nickname** and optionally **Province**.
8. Click **"Submit"**.
9. Verify:
   - Slot changes from amber "claimed" to purple "filled" with waveform bars.
   - Progress bar updates (filled count increases).
   - Recording persists after page refresh (IndexedDB).

**Acceptance:** Audio blob is stored in IndexedDB (`heyin-audio` database, `blobs` store). Project metadata is saved to localStorage.

---

## 4. Delete a Recording

1. On the Project page, hover over a filled voice bubble.
2. A red "×" delete button appears in the top-right corner.
3. Click it.
4. Verify: the slot returns to "empty" state. Audio is removed from IndexedDB.

**Acceptance:** Both localStorage and IndexedDB are cleaned up. Slot is available for re-recording.

---

## 5. Playback (Chorus)

1. On the Project page, ensure at least one slot is filled.
2. Click **"Play Chorus"**.
3. Verify:
   - Audio plays line by line in sequence.
   - The currently-playing lyric line is highlighted (purple ring + pulse).
   - **Pause** stops playback at the current line.
   - **Resume** continues from the paused line.
   - **Stop** resets to the beginning.
4. Results after pause/resume cycle: playback continues correctly.

**Acceptance:** Multi-line playback with real pause/resume. No audio glitches or stuck timers.

---

## 6. Generate Chorus (Export to WAV)

1. On the Project page, ensure some slots are filled.
2. Click **"Generate Chorus"**.
3. Watch the progress indicator (loading → mixing → encoding).
4. Verify:
   - A preview audio player appears with the mixed WAV.
   - **"Download WAV"** downloads the file.
   - **"View Work"** navigates to the work page.
   - **"Re-generate"** creates a new version.

**Acceptance:** Mixed WAV file plays all voices together. Download works.

---

## 7. View Work Page

1. After generating a chorus, click **"View Work"**.
2. Verify:
   - Hero section shows title, song name, participant count, duration.
   - Audio player plays the latest version.
   - **Version selector** shows all generated versions (v1, v2, ...).
   - **Switch between versions** — audio changes accordingly.
   - Participants list shows all contributors.
   - Completion chart visualizes fill progress per line.
   - **"View Project"** navigates back to the source project.

**Acceptance:** Each "Generate Chorus" creates a new version. All versions are playable.

---

## 8. Project Status Control

1. On the Project page, use the status toggle buttons:
   - **"Open for Recording"** — allows new submissions.
   - **"Lock Submissions"** — prevents new claims/recordings.
   - **"Mark Completed"** — finalizes the project.
   - **"Re-open"** — re-enables submissions.
2. Open the Join page for a locked or completed project.
3. Verify: appropriate status gate message is shown.

**Acceptance:** Status transitions work. Join page respects status gates.

---

## 9. Profile & Developer Tools

1. Navigate to `/profile`.
2. Verify:
   - Nickname and province can be edited and saved.
   - **"Reset Guest Identity"** clears guest profile and reloads.
   - **"Clear All Local Data"** (with confirmation) wipes everything.
3. Click **"Developer Tools"** to expand the panel.
4. Use:
   - **Clear Projects** — removes all project/work/version localStorage.
   - **Clear Audio** — deletes IndexedDB audio database.
   - **Reset Guest** — clears guest profile.
   - **Seed Demo Audio** — regenerates demo sine-wave tones.
   - **Export Debug JSON** — downloads all localStorage data as JSON.
   - **Import Debug JSON** — restores data from a previous export.
5. After "Clear All Local Data", verify: home page shows no projects, demo audio is gone until re-seeded.

**Acceptance:** All DevTools operations work without errors. Clear + re-seed cycle is repeatable.

---

## 10. Claim Lifecycle & Stale Cleanup

1. Join page: click an empty slot → it becomes "claimed".
2. **Close the modal without submitting** → slot returns to "empty".
3. Repeat step 1, then **submit** → slot becomes "filled".
4. To test stale claim cleanup:
   - Claim a slot.
   - Close the tab immediately (before the 15-minute timeout).
   - Reopen the project page.
   - After 15 minutes from the claim, the slot is auto-released.

**Acceptance:** Claims are properly released on cancel. Stale claims are cleaned up on project load.

## Storage Summary

| Data | Storage | Key Pattern |
|---|---|---|
| Projects | localStorage | `heyin:project:{id}` |
| Works | localStorage | `heyin:work:{id}` |
| Versions | localStorage | `heyin:version:{id}` |
| Guest Profile | localStorage | `heyin:guest` |
| Audio Blobs | IndexedDB | `heyin-audio` / `blobs` / `{uuid}` |

## Known Limitations (v0.6)

- All data is local to one browser. Real multi-device sharing requires Phase 6 (Supabase).
- Backup track (`backingTrackUrl`) is reserved but not implemented.
- No user authentication — guest identity is a simple localStorage profile.
- Cross-browser sharing is only possible via Export/Import Debug JSON.
