# Heyin v0.8 — Internal Testing Guide

## 测试前准备

### 1. 创建一个测试项目

打开 EdgeOne 部署链接，点 **Create Chorus Project**：

| 字段 | 填什么 |
|---|---|
| Project Title | `十年大合唱（内测）` |
| Song Name | `十年` |
| Lyrics | 用陈奕迅《十年》前 4 句 |
| Voices per Line | 2 |

点 Create。

### 2. 复制 join 链接

在项目页点 **Share** 按钮，复制 join 链接。

### 3. 发给测试者

把下面的消息 + join 链接发给每个测试者。

---

## 发给测试者的消息（复制即用）

---

我做了一个多人合唱 H5 小工具，想请你帮我测一下，**2 分钟就能搞定**。

**步骤：**

1. 打开这个链接 👇
   【粘贴你的 join 链接】

2. 输入一个昵称（比如"小雨"）

3. 点一个**空的歌词槽位**（灰色圆形）

4. 浏览器会问你要麦克风权限 → 点「允许」

5. 录一句歌词 → 点红色「Stop Recording」

6. 试听一下 → 点「Submit」提交

7. 点绿色的已填槽位，听听你刚才录的

**不用唱得好！** 就是帮我测功能。主要看：
- 页面能不能打开
- 麦克风能不能用
- 提交顺不顺
- 有没有地方看不懂

有任何问题直接告诉我！

## Creator 自测清单（你自己跑）

### 项目页

| # | 操作 | 预期 |
|---|---|---|
| C1 | 打开自己的项目页 `/project/{id}` | 页面正常 |
| C2 | 看到已提交的 filled slot | 显示昵称，绿色实心 |
| C3 | 点 filled slot 播放 | 听到声音 |
| C4 | 等所有人提交完 | 至少有 2 个 filled slot |
| C5 | 点 **Play Chorus** | 多轨同时播放 |
| C6 | 点 **Generate Chorus** | 显示 Loading → Ready |
| C7 | 预览 WAV | 项目页能播放 |
| C8 | 点 **View Work** | 跳转 /work/{id} |

### Work 页

| # | 操作 | 预期 |
|---|---|---|
| W1 | 页面加载 | 不报错 |
| W2 | Work 音频播放 | 能听到完整合唱 |
| W3 | 点 **Download WAV** | 下载 |
| W4 | 点 **View Project** | 回到项目页 |
| W5 | 再 Generate 一次 | 创建 v2 |
| W6 | Version 选择器 | v1/v2 正常切换 |
| W7 | 切换后播放不同版本 | 音频变化 |

### 边界情况

| # | 操作 | 预期 |
|---|---|---|
| E1 | 删除一个 slot 录音 | slot 变回 empty |
| E2 | 锁项目（close） | join 页显示 closed |
| E3 | 解锁项目（open） | join 页恢复 |
| E4 | 完成项目（completed） | join 页显示完成 |
| E5 | Delete Project | 项目从列表消失 |
| E6 | 手机 Safari 打开 join | 录音弹窗正常显示 |
| E7 | 微信内打开 join | 基本可用 |

## 已知会有的问题（提前告诉测试者）

- 第一次打开可能需要 2-3 秒加载
- 录音前会弹出麦克风权限，选「允许」就好
- 如果 Submit 报错，关掉弹窗重试一次
- 微信内置浏览器可能会有兼容问题，建议用手机自带浏览器打开
